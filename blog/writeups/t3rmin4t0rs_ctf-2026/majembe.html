🧭 Challenge Overview

We are given the following cryptographic parameters:

  • Two RSA moduli: $N_1, N_2$
  • Two ciphertexts: $ct_1, ct_2$
  • A common public exponent: $e = 65537$

Goal: Recover the flag from the encrypted data.

🔎 Initial Observation & Vulnerability

At first glance, this looks like two independent RSA encryptions. However, a key detail stands out: we have the same exponent ($e$) but two different moduli ($N_1, N_2$).

This raises immediate suspicion of a common vulnerability: ⚠️ The moduli may share a prime factor.

💥 Core Vulnerability: Shared Prime

If two RSA moduli share a prime factor, they can be represented as:

$$N_1 = p \times q_1, \quad N_2 = p \times q_2$$

Because $p$ is a common divisor to both $N_1$ and $N_2$, we can effortlessly recover $p$ using the Greatest Common Divisor (GCD) algorithm:

$$p = \gcd(N_1, N_2)$$

🛠️ Exploitation Steps

🔹 Step 1: Compute GCD

p = gcd(N1, N2)

✅ Result: [+] Shared prime recovered! This confirms both moduli indeed share a common prime.

🔓 Step 2: Factor the Moduli

Now that we have $p$, we can simply divide the moduli to find the other primes:

q1 = N1 // p q2 = N2 // p

🔑 Step 3: Compute Private Keys

We compute Euler's Totient for both keys:

$$\phi(n) = (p - 1)(q - 1)$$
phi1 = (p - 1) * (q1 - 1) phi2 = (p - 1) * (q2 - 1)

Then compute the private exponents $d \equiv e^{-1} \pmod{\phi(n)}$:

d1 = modinv(e, phi1) d2 = modinv(e, phi2)

🔐 Step 4: Decrypt Ciphertexts

Using standard RSA decryption ($m = c^d \bmod n$):

m1 = pow(ct1, d1, N1) m2 = pow(ct2, d2, N2) pt1 = long_to_bytes(m1) pt2 = long_to_bytes(m2)

📄 Step 5: Analyze Plaintexts

Printing the outputs yields:

PT1: b'...\x74\x33rmctf{h4kun4_sh4mb4_ngu' PT2: b'mu_in4y0shind4_j3mb3_69c}...'

🔍 Observation: The flag is split across both plaintexts! We need to reconstruct it by concatenating the two.

🏁 Step 6: Extract Flag

combined = pt1 + pt2 start = combined.find(b"t3rmctf{") end = combined.find(b"}") flag = combined[start:end+1]

🧪 Full Solve Script
