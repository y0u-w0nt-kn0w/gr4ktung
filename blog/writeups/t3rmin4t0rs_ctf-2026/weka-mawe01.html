🧭 Challenge Overview

We are given a service running on nc localhost 9009. Upon connection, we are greeted with the following prompt:

Welcome to t3rmin4t0rs service Enter an order to alphaEcho ^^:

🔎 Initial Testing & Logic Reconstruction

Standard commands like ls, pwd, or id returned "not found" errors, but with a strange twist: the command names were being prepended with the word "echo" without a space.

Input: ls -> Output: sh: 1: echols: not found Input: id -> Output: sh: 1: echoid: not found

🧠 Understanding the Behavior:
It's clear the backend is taking our input and running something like os.system("echo" + input). Because there is no space between "echo" and our input, the shell tries to execute a single concatenated binary name.

🔍 The Vulnerability

Analyzing the backend logic (reconstructed from trial and error), we find a lethal combination of flaws:

  1. Direct Command Injection: The use of os.system() allows us to break out of the "echo" command using a semicolon (;).
  2. Strict Input Truncation: Only the first 3 characters of our input are processed.
  3. Weak Filter: It only blocks the string sh within those 3 characters.
Input Processed Command Status
;lsecho;ls✅ Success (Lists files)
;cat flag.txtecho;ca❌ Failed (Truncated to 3 chars)
;sh[Filtered]❌ Failed (Blocked by filter)

💥 Exploitation: The "Small" Big Brain Move

We know we have RCE, but we are trapped by the 3-character limit. We need a payload that is 3 characters or less but grants us full system access. While ;sh is blocked, there is another legendary interactive binary that fits perfectly: vi.

🚀 Step 1: Triggering Vim

Input: ;vi

This bypasses the filter and fits the 3-character limit. The service now launches the Vim text editor inside our terminal session.

🚀 Step 2: Escaping to Shell

Once inside Vim, we can use the built-in shell escape command. Since the filter is no longer checking our input once the binary is running, we can call sh directly:

:!sh

✔ Shell obtained! We are now operating in a standard shell environment without character limits or filters.

🏁 Flag Recovery
