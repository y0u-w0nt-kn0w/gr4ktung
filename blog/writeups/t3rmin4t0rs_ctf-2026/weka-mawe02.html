Official Writeup: W3k4-m4w3 02

> Published: March 13, 2026 | ⏱ 3 min read | Author: gr4ktung
Howdy!!!, I'm the creator of this challenge in the t3rmin4t0rs CTF 2026. Due to server being down..., I accessed the lab locally for this walkthrough.

🧭 Challenge Overview

We are given a service running on localhost:8001. Upon connecting via Netcat (nc localhost 8001), we are greeted with the following rules:

Welcome to math class!
There are 3 expressions to solve
You have only 5 seconds for each one so be quick ^^

⚠️ The Bottleneck: Manual Speed

Trying to solve these manually is nearly impossible due to the strict 5-second window for complex operations. My manual attempts resulted in either "Wrong answer" or the dreaded "Too slow!" message:

Here is your expression: 837 - 972 / 224 Too slow !

🚨 The Problem: Human input speed and calculation time cannot compete with the server's timeout. This challenge forces us into Automation.

🧠 Solution Strategy

To capture the flag, I needed a script that could:

  1. Establish a TCP connection to the remote service.
  2. Parse the incoming math expression from the stream.
  3. Programmatically evaluate the result.
  4. Send the answer back instantly.

🛠️ Automation with Pwntools

I utilized the pwntools library, which is the gold standard for CTF scripting. The server follows a predictable pattern: it sends "Here is your expression: " followed by the math string, and then expects input after "Your answer: ".

💻 Solve Script

from pwn import * def main(): HOST = "localhost" PORT = 8001 # Establish remote connection p = remote(HOST, PORT) for i in range(3): # Read until the prompt p.recvuntil("expression: ") # Extract the expression line expression = p.recvline().rstrip().decode() # Evaluate the string as math and round to int solution = int(eval(expression)) # Send answer immediately p.sendlineafter("answer: ", str(solution)) # Grab the flag output p.recvuntil("flag: ") flag = p.recvline().rstrip().decode() log.info(f"Flag recovered: {flag}") if __name__ == "__main__": main()

⚙️ Technical Breakdown

  • p.recvuntil("expression: "): This clears the buffer of any introductory text, stopping exactly where the math begins.
  • eval(expression): Python's built-in eval() is perfect for this, as it can handle arithmetic strings natively.
  • p.sendlineafter(...): This ensures we don't send the answer prematurely; it waits for the server's prompt before pushing our solution into the pipe.
