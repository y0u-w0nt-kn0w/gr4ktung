🔍 Initial Recon

Upon visiting the application, we are presented with a page labeled "Leenode Manager".

Clicking the login button simply leads to a page that responds with:

“Authorized personnel only”

At this point, no visible login form is accessible, suggesting the application is relying heavily on hidden endpoints and access control via obscurity.

🔎 Directory Enumeration

Using gobuster, I started enumerating hidden paths on the server:

gobuster dir -u http://127.0.0.1:8004 -w /usr/share/wordlists/dirb/common.txt -x txt,html,php

📁 Discovered endpoints: /admin

Visiting /admin still showed the "Authorized personnel only" message, so I ran a second scan specifically on that directory:

gobuster dir -u http://127.0.0.1:8004/admin -w /usr/share/wordlists/dirb/common.txt -x txt,html,php

This yielded two very interesting files: /todo.txt and /todo.

📄 Analyzing /admin/todo.txt

Accessing the TODO file reveals the developer's notes:

TODO – Admin notes - Hide real login page name (done) - Rename panel page before production (p4n3l.html) - Remove legacy auth reference: 70g1n.html - Disable directory listing

🧠 Key insights:

  • Login page: 70g1n.html
  • Admin panel: p4n3l.html
  • The developer relied entirely on security through obscurity.

🚫 Access Control Behavior

Trying to access /admin/p4n3l.html directly redirects us to /admin/70g1n.html. This indicates that session-based authentication is actively required to view the panel.

🔐 Targeting .htpasswd & Exploiting Double URL Decoding

Given the hint and typical Apache setups, the .htpasswd file is a strong target. However, direct access to /.htpasswd returns a 403 Forbidden.

From analyzing the backend logic behavior, it appears the server decodes the URL twice. It checks and blocks direct access like this:

if decoded_once in ["/.htpasswd", "/.htaccess"]

But it fails to block "../.htpasswd" in decoded_twice! We can bypass this filter using a Double URL Encoding technique.

🔓 Bypass technique:

Stage Value
Raw Payload%252e%252e%252f.htpasswd
Decode Once (Bypasses Filter)%2e%2e%2f.htpasswd
Decode Twice (Executed)../.htpasswd ✅

🚀 Exploit:

curl http://127.0.0.1:8004/%252e%252e%252f.htpasswd

📥 Result:

t3min4tors_1een0de:nlGc3XNhkrL1o

🔑 Cracking the Password

The extracted hash is nlGc3XNhkrL1o. Because it is exactly 13 characters long, it indicates a DES crypt hash, which is very common in legacy .htpasswd files.

🔓 Using John the Ripper:

echo 't3min4tors_1een0de:nlGc3XNhkrL1o' > hash.txt john --format=descrypt hash.txt john --show hash.txt

✅ Cracked credentials:

  • Username: t3min4tors_1een0de
  • Password: ktw2z

🏁 Gaining Access

Finally, we navigate to the hidden login page at /admin/70g1n.html and authenticate using our cracked credentials. After successful authentication, we are automatically redirected to the admin panel at /admin/p4n3l.html.
